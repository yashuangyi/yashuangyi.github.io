Linux简介

  1. 简介

    • Linux是操作系统
    • Linux沿袭了Unix的良好特性
    • Linux就是内核与系统调用接口这两层
    • 是云计算和大数据平台的基石

  2. 主要特点

    • 系统稳定
    • 免费
    • 安全性高
    • 多任务、多用户
    • 不耗资源
    • 适合嵌入式系统

  3. Linux内核的功能

    • 进程调度
    • 存储管理
    • 虚拟文件系统
    • 网络接口
    • 进程通信

  4. Linux内核的版本

    • 内核版本号格式为：a.bb.cc(a是主版本号，bb是次版本号，cc是修订版本号)
    • bb为偶数时表示稳定版，为奇数时表示开发测试版
    • 内核文件名字是vmlinuz+版本号
    • 内核文件存放在目录/boot中

  5. 引导和安装

    • Linux引导装载程序一般安装在硬盘的MBR
    • /etc/inittab可以设置开机直接进入图形化界面
    • swap分区是交换分区，作为虚拟内存使用，等于1倍或2倍的物理内存
    • 常用引导装载程序有
      1. grub
      2. lilo

  6. Linux发行版的主要分支

    • RedHat，代表作：RHEL
    • Slackware，代表作：SUSE
    • Debian，代表作：Ubuntu

  6. 运行级别

    • 定义：操作系统当前正在运行的功能级别
    • Linux定义了0~6的7种运行级别，定义在/etc/inittab文件中
    • 7钟运行级别如下：
      • 0 halt 关机，代表系统停机状态
      • 1 Single user mode 单用户模式，只支持root账户（类似Win的安全模式）
      • 2 Multiuser，without NFS 不支持网络文件系统的多用户模式
      • 3 Full Multiuser mode 完全多用户模式（默认不启动X Window图形界面服务）
      • 4 unused 系统未使用，用作保留
      • 5 X11 图形界面的多用户模式（默认启动X Window图形界面服务）（CentOS开机默认进入）
      • 6 Reboot 重启
    • 原理
      1. 在目录/etc/rc.d/init.d下有许多服务器脚本程序，一般称为服务（service）
      2. 在/etc/rc.d下有7个名为rcN.d（N取值为0~6）的目录，对应系统地7个运行级别
      3. rcN.d目录下是一些符号链接文件，指向init.d目录下的service脚本文件，命名规则为K+nn+服务名或S+nn+服务名(nn为两位数字)
      4. 系统根据指定的运行级别进入对应的rcN.d目录，并按照文件名顺序检索目录下的链接文件：对于K（Kill）开头文件，系统将终止其对应服务；对于S（Start）开头文件，系统将启动其对应服务
      5. CentOs默认情况下主要有两种targets：multi-user.target和graphical.target
    • 相关命令
      1. runlevel：查看运行级别
      2. init N：进入其它运行级别（N取0~6）

Linux的文件类型

  1. 文件类型及符号

    • 普通文件：-
    • 目录文件：directory：d
    • 字符设备文件：character：c
    • 块设备文件：block：：b
    • 套接字文件：socket：s
    • 命名管道文件：pipe：p
    • 符号链接文件：link：l

  2. find命令查找指定类型的文件

    find / -type [f d b c s p l] [-ls] [|wc-l]：[-ls]可以显示找出的文件的详细信息，[| wc -l]管道，可统计数量

  3. 文件命名规则

    • 除字符/外，所有字符都可使用
    • 长度不超过255个字符
    • 若文件名的第一个字符为.，表示该文件为隐藏文件
    • 区分大小写
    • 文件属性与取名无关

  4. 目录、路径基础

    • 目录：指包含许多文件项目的一类特殊文件。分为：父目录、子目录、工作目录、用户主目录
    • .表示当前目录，..表示父目录，~表示用户主目录
    • 绝对路径：由根目录/开始写起
    • 相对路径：不是由根目录/写起，而是由当前目录开始

  5. 文件、目录权限管理

    • 三类用户：

      1. 文件所有者owner：也就是文件创建者：rwx
      2. 文件的同组用户group：同组用户对属于该组的文件有相同的访问权限：rw-
      3. 其他用户others：既非以上两种用户，也非用户root：r-x

    • 权限表示

      1. 字母表示法：rwx对应于所有者的权限，即可读可写可执行
      2. 数字表示法

    • 可输入ls-l或ll命令查看文件权限

    • chattr用于设置文件的隐藏权限，lsattr用于显示文件的隐藏权限

    • 由于系统默认屏蔽（root的umask）的权限为022（普通用户则为002），因此新创建的目录权限就为777-022=755，新创建的普通文件权限为666-022=644。

    • 改变文件所有权

      1. chown：改变文件所有权：chown [OPTION] ...[OWNER] [:GROUP] FILE ...

      2. chgrp：改变文件所在组：chgrp :[选项] [组名] [文件名]

      3. chmod：数字修改法（chmod n1n2n3 <文件|目录>）、字母修改法（加减权限）

        [ugoa...][[+-=][rwxX]...][,...]

        • u 表示该文件的拥有者，g 表示与该文件的拥有者属于同一个群体(group)者，o 表示其他以外的人，a 表示这三者皆是
        • r 表示可读取，w 表示可写入，x 表示可执行，X 表示只有当该文件是个子目录或者该文件已经被设定过为可执行
        • + 表示增加权限、- 表示取消权限、= 表示唯一设定权限

      4. 改变文件夹内所有文件所有权时要加-R递归

    • 特殊权限

      1. s权限（setuid）：SUID：让普通用户可以以root用户角色运行只有root账户才能运行的程序或命令：4---
      2. s权限（setgid）：SGIT：任何用户在此目录下创建的文件都具有和该目录所属的组相同的组：2---
      3. t权限（sticky bit）：SBIT：防删除位：用户能添加文件但不能删除：1---

  6. 文件归档与压缩

    • 压缩/解压缩命令：压缩后的后缀名：打开压缩文件的命令

      1. gzip/gunzip(gzip -d)：*.gz：zcat
      2. compress/uncompress：*.z：zcat
      3. bzip2/bunzip2(bzip2 -d)：*.bz2：bzcat

    • 先归档再压缩：.tar：打包但未压缩的文件

    • 归档命令：tar <选项 备份后的文件名 源文件或目录：为文件和目录做备份，归档为tar文件

  7. 软链接：类似于windows的快捷方式：ln -s [目标目录] [软链接地址]

Linux中的硬盘管理

  1. 硬盘一般分为IDE硬盘、SCSI硬盘和SATA硬盘

    • 在Linux中，IDE接口的设备被称为hd
    • SCSI和SATA接口的设备则被称为sd。第1块硬盘被称作sda，第2块被称作sdb…以此类推
    • Linux规定，一块硬盘上只能存在4个主分区（或3个主分区+1个扩展分区），分别被命名为sda1、sda2、sda3和sda4（此时不能再创建扩展分区）
    • 一块硬盘最多只能存在1个扩展分区，扩展分区上可新建逻辑分区
    • 逻辑分区从5开始标识，每多一个逻辑分区，就在末尾的分区号上加1。逻辑分区没有个数限制

  2. 硬盘分区及格式化

    • 查看电脑硬盘信息命令：ls /dev | grep sd
    • 进入指定硬盘：fdisk /dev/硬盘名
    • 在指定硬盘上新建分区：new
    • 操作完后记得输入w保存写入
    • 格式化分区：mkfs -t 文件格式 /dev/分区名（否则无法打开分区）
    • 修改完/etc/fstab文件后务必使用mount -a命令测试有没有错误
    • 挂载磁盘到根目录某空文件夹：（否则无法打开分区）
      1. 先在根目录新建空文件夹：mkdir /名称
      2. 将分区映射到该空文件夹：mount /dev/分区名 文件夹
    • 自动挂载文件系统（否则每次开机都要重新挂载）：OS启动后会自动挂载磁盘，在/etc/fstab文件中存储了自动挂载文件系统的参数
      1. 可在/etc/fstab文件尾部添加/dev/分区名 /web 文件格式 defaults 0 0：gedit /etc/fstab
      2. 保存后重启
      3. 系统启动后就可以通过/web目录直接访问相应分区中的内容，而不用每次都使用mount命令来挂载该分区

  3. 磁盘限额

    • 磁盘限额是系统管理员用来监控和限制用户或组对磁盘空间的使用情况的工具；它可以保证所有用户都拥有自己独立的文件系统空间，确保用户使用系统空间的公平性和安全性

    • 使用磁盘限额的必须条件

      1. 磁盘限额是针对分区的，只有分区才有磁盘限额的说法
      2. 需要内核的支持
      3. 分区格式必须是ext4的文件系统才行
      4. 只有管理员的权限才能设置磁盘限额
      5. 磁盘限额只能限制普通用户，对root用户无效

    • 基本概念

      1. 硬限制：指的是每个用户或组可拥有的磁盘空间或文件的绝对数量，绝对不允许超过这个限制
      2. 软限制：一个用户在文件系统可拥有的最大磁盘空间和最多文件数量，在某个宽限期内可以暂时超过这个限制
      3. 宽限时间：默认为7天。如果7天后，用户使用磁盘空间的数量仍然超出软限制，则系统将会禁止此用户账号。要重新激活账号，必须由系统管理员来进行

    • 限制方面

      1. 限制用户或组可以拥有的文件数，每个文件都有一定的大小
      2. 限制分配给用户或组的磁盘块的数目

    • 相关命令

    • 通过将硬盘中某一个可用分区的类型转换为Linux swap/ Solaris分区，然后制作成swap交换分区，就可以增加Linux系统内的虚拟内存

用户账户和组帐户管理

  1. 用户账户管理

    • Linux使用用户权限机制对系统进行管理

      1. 主要功能：提供不同用户使用本系统的权限分配
      2. 目的：保证用户数据与文件的安全

    • 账号实质上就是一个用户在系统上的标识，系统依据账户来区分每个用户的文件、进程、任务，给每个用户提供特定的工作环境

    • Linux的三类用户

      1. 超级用户（root用户）

        每个Linux系统都必须有，并且只有一个。它拥有最高的权限，可以删除、终止任何程序

      2. 系统用户

        与系统运行和系统提供的服务密切相关的用户，通常在安装相关的软件包时自动创建并保持默认状态。系统用户不能登录计算机

      3. 普通用户

        在系统安装后由超级用户创建的，通常完成指定权限的操作，而且也只能操作自己所拥有权限的文件和目录

    • 用户账户文件/etc/passwd组成——存储用户账户信息

      1. 用户名：在系统中时唯一的，可由字母、数字和符合组成
      2. 口令：此字段用“x”代替，而将口令保存在/etc/shadow文件中
      3. 用户ID（UID）：系统内部用它来标识用户且唯一
        • 超级用户：UID=0,GID=0
        • 系统用户：0<UID<1000
        • 普通用户：UI>=1000
      4. 组ID（GID）：系统内部用它来标识组属性
      5. 用户相关信息：例如用户全名等
      6. 用户主目录：用户登录系统后所进入的目录
      7. 用户登录环境：用户第一次登录Shell环境

    • 用户影子文件：/etc/shadow存储关于账户口令相关设置——只有root才有权限进行修改，普通用户不能读取该文件

      1. 用户名：用户登录到系统时使用的名字，而且是唯一的
      2. 口令：存放MD5加密过的口令
      3. 标识从1970年1月1日到用户最近一次修改口令所经过的天数
      4. 口令保持有效的最小天数，即口令在多少天内不能被用户修改
      5. 口令在多少天后需要被修改
      6. 口令到期前多少天内给用户发出警告
      7. 口令过期多少天后口令失效

  2. 组帐户管理

    • 组管理

      1. 组是具有相同特性的用户集合
      2. 设置的目的主要是便于权限的统一组织和分配

    • 组按性质划分为系统组和私有组账户

      1. 系统组：安装Linux及部分服务性程序时系统自动设置的组
      2. 私有组：系统安装完成后，由超级用户新建的组

    • 组管理文件构成

      1. 组帐户文件：/etc/group
        • 组名：组的名称，由字母、数字和符号组成
        • 组ID（GID）：识别不同组的唯一标识
        • 组口令：默认情况下不使用，必须经过特殊设置
        • 组内用户列表：属于该组的所有用户名表，用“，”间隔
      2. 组影子文件：/etc/gshadow
        • 同上

    • 组帐户管理命令

      1. groupadd命令

        格式：groupadd [-g GID] <组帐户名>

        功能：新建组群，只有超级用户才能使用此命令。该指令的执行将在/etc/group文件和etc/gshadow文件中分别增加一行记录

        选项：

        • -g：用于指定GID；不加的话则使用系统默认GID
        • -r：用于创建系统组账号（GID小于1000）

      2. groupmod命令

        格式：groupmod [参数] <组帐户名>

        功能：修改指定组群，只有超级用户才能使用此命令。该指令的执行将修改/etc/group文件相应记录的内容

        选项：

        • -g：改变组帐户的GID；组帐户名保存不变
        • -n：改变组帐户名

      3. groupdel命令

        格式：groupdel <组帐户名>

        功能：删除指定组群，只有超级用户才能使用此命令

        注意：

        • 被删除的组帐户必须存在
        • 当有用户使用组帐户作为私有组时不能删除
        • 与用户名同名的私有组账户在使用userdel命令删除用户时被同时删除

系统管理

  1. 图形界面管理
    • X-Window
      1. X-Server（X服务器）：处理来自X-客户机的请求
      2. X-Client（X客户机）：任何一个应用程序都是X-客户机
      3. X-Protocol（X通信协议）：是X-服务器与X-客户机之间的通信协议
    • X-Window的特点
      1. 良好的网络支持
      2. 个性化的窗口界面
      3. 跨平台特性
      4. 开源软件
    • GNOME图形环境：是一种流行的Linux桌面环境
  2. 进程管理
    • 进程分类
      1. 交互进程：由shell启动的进程，既可以在前台运行，也可以在后台运行
      2. 批处理进程：是一个进程序列，该进程负责按照顺序启动其它进程
      3. 守护进程：指执行特定功能或者执行系统相关任务的后台进程
    • 启动前台进程：用户在shell输入一条命令，即可启动一个前台进程
    • 启动后台进程：只需在命令后面添加一个字符“&”
    • 查看进程状态的命令
      1. uptime：查看过去的1、5、15分钟内运行队列中的平均进程数量
      2. ps：监控系统后台进程的工作情况
      3. top：显示进程所占系统资源；默认每隔3秒刷新一次
    • 终止前台进程
      1. 在程序退出时，该前台进程将自动结束
      2. 在程序运行过程中，按ctrl+c退出前台进程
    • 终止后台进程：使用kill命令
      1. kill -15 pid——正常结束一个进程
      2. kill -9 pid——强制终止一个进程
  3. 软件包管理
    • CentOS安装软件经常使用的软件包管理器为RPM和YUM
    • RPM：Redhat Package Manager
      1. 缺点
        • 安装的环境必须与编译时的环境一致或相当
        • 包与包之间可能存在着相互依赖的情况
        • 卸载包前需要先把依赖包卸载
      2. 命名格式：name-version-release.arch.rpm
      3. 优点
        • 可以安装、删除、升级和管理软件
        • 可以把自己做的程序包打包成rpm包发布
        • 能利用签名GPG和MD5检测软件包的来源安全性和内容完整性
      4. 使用权限
        • RPM软件的安装、删除、更新只有root权限才能使用
        • 对于查询功能任何用户都可以操作
        • 如果普通用户拥有安装目录的权限，也可以进行安装
      5. 常用命令
        • rpm -ivh 文件名：安装RPM包
        • rpm -Uvh 文件名：升级RPM包
        • rpm -q rpm 包名：查询一个包是否安装
        • rpm -e 文件名：卸载RPM包
        • rpm -qi 包名：得到一个已安装rpm包的相关信息
        • rpm -ql 包名：列出一个rpm包安装的文件
        • rpm -qf 文件的绝对路径：列出某一个文件属于哪个rpm包
    • yum
      1. 安装软件：yum install vstfpd
      2. 删除软件：yum remove vsftpd
      3. 查询软件
        • 搜寻与磁盘阵列相关的软件：yum search raid
        • 找出mdadm这个软件的功能如何：yum info mdadm
        • 列出yum服务器上面提供的所有软件名称：yum list
        • 列出目前服务器上可供本机进行升级的软件：yum list updates
        • 列出提供passwd这个文件的软件有哪些：yum provides passwd
      4. 查询软件组：yum grouplist
      5. 安装软件组：yum groupinstall Eclipse
      6. 删除软件组：yum groupremove Eclipse
  4. 网络管理（网络环境配置及测试）
    • 通过主机名可以实现局域网内的访问
      1. 查看主机名：hostname
      2. 设置主机名：hostname 新名字(临时生效)
      3. 在文件中修改主机名：vi /etc/hostname(永久生效)
    • 使用ifconfig配置网卡
      1. 查看所有网卡信息：ifconfig
      2. 查看指定网卡的情况：ifconfig eth0
      3. 配置ip地址：ifconfig 网卡名 ip地址 netmask 子网掩码
      4. 禁用网卡：ifconfig 网卡名 down
      5. 启用网卡：ifconfig 网卡名 up
    • ping命令检测到特定主机的网络连通性：ping [可选项] ip/域名/主机名

Shell编程入门

  1. Shell的编程过程

    • Shell独立于内核，它是连接内核和应用程序的桥梁，并由输入设备读取命令，再将其转为计算机可以理解的机械码，Linux内核才能执行该命令
    • Shell不但能与用户进行逐个命令交互执行，也可以通过脚本语言进行编程
    • 通过对Shell进行编程，可使大量任务自动化，对于系统管理、系统维护方面特别有用
    • Shell脚本通常都是以.sh为后缀
    • 第一行必须以#! /bin/bash开头，意思是该文件使用的是bash语法
    • Shell脚本的运行
      1. 赋予权限，直接执行：
        • chmod a+x first.sh
        • ./first.sh
      2. 使用Shell程序执行：sh first.sh
      3. Shell的调试执行：sh -x first.sh
    • 查看系统内置了哪些Shell
      1. cat /etc/shells
      2. chsh -list

  2. Shell的语法介绍

    • Shell的环境变量：PATH、SHELL、HOME

    • 使用变量：在Shell程序中所有变量中保存的值都是字符串

    • 常用的预定义变量

      1. $#：传递到脚本或函数的参数数量
      2. $*：传递到脚本或函数的全部参数
      3. $?：前一个命令执行情况，返回0表示成功，其他值表示失败
      4. $$：当前进程的ID（PID）
      5. $!：后台运行的最后一个进程的进程ID（PID）
      6. $0：当前脚本的名称

    • 输入输出命令

      1. 输出命令：echo
      2. 输入命令：read

  3. 调试Shell程序

    • 常见的一般错误
      1. 输入错误
      2. 字符大小写错误：Linux严格区分大小写，所有关键字都用小写字母表示，建议变量名用大写字母组合表示
      3. 循环错误
    • 调试跟踪
      1. -n选项：不执行脚本，仅检查脚本中的语法问题
      2. -v选项：将执行过的每一个脚本命令都原样输出到终端
      3. -x选项：在执行过的每个命令的行首添加“+”号显示在终端，并显示变量的值。使用该选项更方便跟踪程序的执行过程
