Docker安装及基本命令

前言

距离上次使用docker已经有一年多了，只不过之前的使用大多都是照葫芦画瓢，并没掌握其原理。正好最近学校大数据课程要求我们搭建hadoop环境（一个Master、两个Slave节点），为了方便配置环境我决定使用docker来搭建。由于之前买的书还在学校宿舍= =我便根据网上教程自行整理一遍好了~

注：以下演示环境为Win10系统，其它系统用户请参考教程：https://www.runoob.com/docker/windows-docker-install.html

简介

曾经的我也会有这个疑问：Docker是用来干嘛的？不是已经有虚拟机了吗？

Docker可以说是一个终端命令行的虚拟机，或者说是一个虚拟环境，亦或者说它是容器，但又不只是个容器，它包含3个重要概念：

  1. 镜像（Image）

    镜像是静态的、可被用户互相分享的文件。譬如装系统或虚拟机，都是需要iso镜像才能进行安装，docker中的镜像也是如此，是静态的，用户只能pull别人的镜像或push自己的镜像

  2. 容器（Container）

    容器可以理解为镜像的动态状态，用户可对其进行操作，但关机后将失去修改内容，因此若要保存修改则需要将当前容器封装成一个新的镜像

  3. 仓库（Repository）

    类比Git的代码仓库，用户可pull别人push到公共仓库的镜像到本地，亦可push自己做好环境的镜像上去

安装docker

  1. 检查电脑是否支持虚拟化

    打开任务管理器，点开“性能”一栏，查看CPU信息中是否已启用虚拟化

  2. 安装版本

    需要特别注意的是：Docker支持Win10，但仅支持Win10专业版（一年前踩过此坑，印象深刻…）。以我的Win10家庭中文版为例，须安装DockerToolBox（下载链接：http://mirrors.aliyun.com/docker-toolbox/windows/）

  3. 注册Docker账号

    链接：https://hub.docker.com/signup

    注册后便可像github般上传项目到仓库中

  4. 注册DaoCloud

    大多数时候直接从docker官方往本地pull镜像会十分缓慢，因此可通过国内docker服务提供商免费获取加速pull镜像服务，我这里用的是DaoCloud（链接：https://www.daocloud.io/）

    注册完毕后在其首页按教程获取加速服务

  5. 注意Git路径

    需要注意的是，若Git的安装路径不是默认的C盘的话，打开docker将会提示找不到“bash.exe”，这时需修改快捷方式的路径，参考：https://blog.csdn.net/pingfan2014/article/details/81145813

  6. 启动docker

    若启动docker时提示“No default Boot2Docker ISO found…”，请参考：https://blog.csdn.net/dhaiuda/article/details/81975550，亲测有效

  7. 使用SecureCRT登录

    default虚拟机的默认用户名为docker，密码为tcuser，ip为docker主界面所显示

  8. 更改虚拟磁盘存储位置

    虚拟机的默认存储位置是C盘，为了避免后期C盘爆红（我亲自体验过一次，因而买了新固态重装了电脑…）最好将存储位置改为其它盘

  9. 如果使用的是boot2docker这一发行版，首先执行docker-machine ssh default进入容器中，然后执行sudo -i进入root用户，这样才能执行sudo命令

  10. Hadoop集群搭建教程：https://kiwenlau.com/2016/06/12/160612-hadoop-cluster-docker-update/

    这位大佬写的很好，只不过我目前也仅局限于照着使用，有机会得研究一下他是怎么实现的…

基本命令

  1. 查看docker版本信息

    • docker version 或docker -v

  2. 获取镜像

    • docker pull 镜像名，例如：docker pull ubuntu:latest

  3. 搜索镜像

    • docker search 镜像名，例如：docker search ubuntu

  4. 查看本地镜像仓库

    • docker images
    • 可见字段信息如下：
      1. REPOSITORY：来自哪个仓库
      2. TAG：镜像的标记
      3. IMAGE ID：镜像ID号，唯一
      4. CREATED：创建时间
      5. SIZE：镜像大小

  5. 启动新镜像

    • docker run -it 仓库名 命令解析器，例如：docker run -it chug/ubuntu14.04x64 /bin/bash
    • 说明：
      1. -it表示运行在交互模式，是-i -t的缩写，-i表示打开并保持stdout，-t表示分配一个终端。即启动bash，和容器有命令行的交互
      2. 当终端字头变成例如root@aa97ba3292ce:/#时，说明镜像已创建一个容器实例，接下来便可对该实例进行操作

  6. 查看当前正在运行的容器

    • docker ps

  7. 操作镜像

    • docker start 容器ID号：启动容器
    • docker stop 容器ID号：停止容器
    • docker restart 容器ID号：重启容器
    • docker attach 容器ID号：进入一个后台启动的容器
    • docker commit -m "说明信息" -a "用户信息" 容器ID号 用户名/仓库名:tag信息，例如：docker commit -m "ubuntu with vim" -a "sgy" aa97ba3292ce sgy/ubuntu:vim：保存容器的更改
    • exit：退出容器

  8. 删除容器或镜像

    • docker rm 容器ID号：删除容器
    • docker rmi 镜像ID号：删除镜像
    • 注意：删除镜像前必须先删除以此镜像为基础的容器
