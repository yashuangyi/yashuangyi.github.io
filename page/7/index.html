Git常用命令

Git的基本命令

  1. 安装Git后的配置（自报家门）

    • git config --global user.name "Your Name"
    • git config --global user.email "email@example.com"

  2. 创建版本库

    • git init

  3. 把文件添加到版本库

    • git add <file> （把文件添加到仓库）
    • git commit -m "本次提交说明" (把文件提交到仓库)

  4. 查看仓库当前状态

    • git status (查看工作区，是否有文件被修改)
    • git diff (查看修改内容)

  5. 查看提交日志

    • git log --pretty=oneline

  6. 回退到某一个版本

    • git reset --hard HEAD^ (HEAD^表示上一个版本，HEAD~100表示上100个版本，也可以用commit id)
    • git revert HEAD 撤销更改并分享给别人

  7. 查看命令记录

    • git reflog

  8. 撤销工作区的修改

    • git checkout -- <file>
    • git reset HEAD <file> （HEAD表示最新的版本）

  9. 删除文件

    • rm <file> （删除本地工作区的文件）

    • git rm <file> （从版本库删除该文件）并且执行git commit -m "本次记录的说明"

    • git checkout --<file> （当误删时，恢复文件）

远程仓库的使用

  1. 创建SSH Key

    • 在用户主目录下找到.ssh目录，在此目录下查看是否有id_rsa以及id_rsa.pub这俩文件，若有则跳过下一步：
    • ssh-keygen -t -rsa -C "youremail@example.com" （创建SSH Key）
    • id_rsa是私钥，不可泄露；id_rsa.pub是公钥，将其内容复制粘贴到GitHub里的SSH Keys页面中

  2. 本地库关联远程仓库

    • git remote add origin git@github.com:用户名/仓库名.git

  3. 把本地库所有内容推送到远程库

    • git push -u origin <当前分支名（默认master）> （之后的推送可省略-u命令）

  4. 克隆远程库到本地库

    • git clone git@github.com:用户名/仓库名.git （SSH协议）

  5. 查看远程库信息

    • git remote

    • git remote -v 查看详细信息

分支管理

  1. 创建并切换分支

    • git checkout -b <分支名> 相当于git branch <分支名> 和 git checkout <分支名>两条命令

  2. 查看当前分支

    • git branch

  3. 合并分支

    • git merge <待合并分支名> 合并指定分支到当前分支（可能会用Fast forward模式）
    • git merge --no-ff <待合并分支名> 禁用Fast forward的合并

  4. 删除分支

    • git branch -d <分支名>
    • git branch -D <分支名> 强制删除（当该分支未被合并时）

  5. 切换分支

    • git switch -c <分支名> 创建并切换到该分支
    • git switch <分支名> 直接切换到已有的分支

  6. 强制修改分支位置

    • git branch -f <分支名> <提交名（例如HEAD^、HEAD~3）> 将指定分支强制指向指定提交

  7. 查看分支合并图

    • git log --gragh

  8. 保存工作区

    • git stash

  9. 查看保存的工作区

    • git stash list

  10. 恢复工作区

    • git stash apply 恢复后不删除stash内容，要用git stash drop删除
    • git stash pop 恢复的同时删除stash内容

  11. 复制一个特定的提交到当前分支

    • git cherry-pick <提交编号>

  12. 分支的管理规则

    • master分支是主分支，要时刻与远程库同步
    • dev分支是开发分支，团队所有成员都需要在上面工作，所以也需要与远程同步
    • bug分支只用于在本地修复bug，没必要推到远程
    • feature分支用于开发新功能

  13. 抓取最新提交

    • git pull 把最新的提交抓下来，以便在本地库合并然后解决冲突
    • git fetch 把最新的提交抓下来，不进行合并

  14. rebase操作

    • git rebase 将分叉的提交变成直线

  15. HEAD

    • HEAD 是一个对当前检出记录的符合引用——也就是指向正在其基础上进行工作的提交记录（通常指向分支名）

标签管理

  1. 创建标签

    • git tag <标签名> 默认绑定HEAD

  2. 查看所有标签

    • git tag

  3. 对某次提交打标签

    • git tag <标签名> <提交编号>
    • git tag -a <标签名> -m <提交编号> -a指定标签名，-m指定说明文字

  4. 查看标签信息

    • git show <标签名>

  5. 删除标签

    • git tag -d <标签名>

  6. 推送标签到远程

    • git push origin <标签名> 推送某个标签

    • git push origin --tags 一次性推送全部尚未推送的标签
